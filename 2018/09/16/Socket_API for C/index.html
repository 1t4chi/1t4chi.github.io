C 语言下 Socket 常用接口

发表于 2018-09-16 | 更新于 2019-03-14 | 分类于 Socket | 评论数： | 阅读次数：
本文详细记录了 C 语言下的 20 个 Socket 常用接口,包括创建、绑定、监听、连接、读写、地址获取等等。更多程序实例请移步《Socket 的 TCP 模型及实现》或 《Socket 的 UDP 模型及实现》

socket

参数

  • family: 协议簇 AF_INET,AF_UNIX
  • type:
    • SOCK_STREAM:TCP
    • SOCK_DGRAM:UDP
    • SOCK_RAW:IP
  • protocol: 默认为0

返回值

  • 大于0(socket描述符)：成功
  • -1：失败

示例

说明

并不是上面的 type 和 protocol 可以随意组合的。如 SOCK_STREAM 不可以跟IPPROTO_UDP组合，当 protocol 为0时，会自动选择 type 类型对应的默认协议。

bind

参数

  • sockfd：socket 描述符
  • myaddr 一个const struct sockaddr
    • * 指针，指向要绑定给 sockfd 的协议地址
    • 这个地址结构根据地址创建 socket 时的地址协议族的不同而不同
  • addrlen 地址长度

返回值

  • 大于0：成功
  • -1：失败

示例：

说明：

不会产生阻塞
客户端与服务器都可以绑定 socket 地址。但通常服务器需要，客户端不需要。不绑定地址时，自动分配端口+本机地址填充 socket 地址。

listen

示例

说明

用于 TCP 的通信，只能应用于面向连接的 socket SOCK_STREAM
执行 listen 后 socket 转换成被动 socket，可以接受连接

请求队列

当一个请求到达时，被插入请求队列。服务器用 accept 函数从队列中移走并响应请求。注意请求队列中的连接已经被TCP接受(即三次握手已经完成)，但还没有被应用层所接受。注意区分 TCP 接受一个连接是将其放入这个队列，而应用层接受连接是将其从该队列移出
对于新的连接请求，如果连接请求队列中已没有空间，TCP 将不理会收到的 SYN，也不发回任何报文段(即不回送RST)。

  • 为什么不回送 RST 呢？
    因为这是一个软错误，通常队列满是由于应用程序或操作系统忙造成的,这个条件在很短的时间内可以改变
    如果回送 RST 会将客户进程的主动打开被废弃,不如让它重复发送 SYN 效率高。

connect

参数

  • sockfd－socket描述符
  • servaddr－服务器地址
  • addrlen－地址结构长度

返回值

  • 0：成功
  • -1：失败
  • errno：错误代码

示例

说明

用于 TCP 的通信；客户机一般不指定自己的端口号，由系统分配一个临时端口；填入的地址是远程服务器的地址；对一个 socket 描述符不能两次使用 connect 函数。

accept

参数

  • sockfd 描述符
  • clientaddr 客户端地址
  • addrlen 地址结构长度

返回值

  • 大于0：新的 socket 描述符标识已接受的连接
  • -1：没有连接请求到达

示例

说明

accept 函数在无连接请求时将阻塞进程，并返回-1。

用于通信的套接字描述符

accept 返回的描述符是真正可以和客户端通信的 socket。
服务器的侦听 socket 只负责侦听和接受连接,不负责通信。

  • accept 新创建的 socket 的端口号和原 sockfd 的端口号相同吗？
    相同！四元组区分 TCP 连接；主监听套接字和新创建的套接字状态不同（listen/established)
    处于 established 的套接字不能接收SYN报文段，而处于 listen 的套接字不能接收数据报文段。

read

参数

  • fd 描述符
  • buf 数据缓冲区
  • len 读取数据大小

返回值

  • 无数据：阻塞
  • 套接字缓冲区>=len：读出len个字节
  • n>0 and n < len：读出n个字节
  • n=0：读通道已关闭
  • n<0：出错或异常，errno为错误代码
  • n=-1,errno==EINTR：由中断引起错误
  • n=-1,errno==ECONNREST：网络连接有问题

说明

  • 一般用于 TCP 的通信
  • 跨越用户态内核态

接收缓冲区

buf 指的是接收或发送进程的应用缓冲区。每个 TCP 套接字都有一个套接字接收缓冲区和发送缓冲区(内核中)，分别用于存放即将从网络读或写向网络的数据。read 并不是从网络读取数据，而只是完成从套接字接收缓冲区到接收进程应用缓冲区复制数据的任务。真正从套接字缓冲区到网络读写数据的过程由内核TCP协议来完成，不需要应用程序干涉，系统调用层面不会给予应用程序任何保证和通知。read/write 跨越了用户态和内核态，本质是读写系统（内核）缓冲区。

write

参数

描述符
数据缓冲区
读取数据大小

返回值

  • 无缓冲区：阻塞
  • 套接字缓冲区 >=len：写入 len 个字节
  • n>0 and n < len：写入 n 个字节
  • n=0：写通道已关闭
  • n<0：出错或异常，errno为错误代码
  • n=-1,errno=EINTR：由中断引起错误
  • n=-1,errno=EPIPE：网络连接有问题

说明

一般用于 TCP 的通信。跨越用户态内核态。write 函数返回成功不代表数据已经发送至对方主机，只说明已经存在发送缓冲区中，等待TCP协议来发送。

send

flag 参数

  • flags=0, send与write功能相同
  • flags=MSG_DONTROUTE，告诉内核目标主机在本地网络，无需查找路由表。适用于局域网或同一网段
  • flags=MSG_OOB，发送带外数据
  • flags=MSG_DONTWAIT，如果套接字缓冲区没有足够空间，则进程不阻塞等待

说明

既可以用于 TCP 也可以 UDP，但个人感觉用于 TCP 较好，需先 connect。比 write 功能强大，拥有 flag 参数。

sendto

flag 参数

  • flags=0, send与write功能相同
  • flags=MSG_DONTROUTE，告诉内核目标主机在本地网络，无需查找路由表。适用于局域网或同一网段
  • flags=MSG_OOB，发送带外数据
  • flags=MSG_DONTWAIT，如果套接字缓冲区没有足够空间，则进程不阻塞等待

说明

只能用于 UDP 的无连接通讯，需指定目的地址

recv

flag 参数

  • flags=0, recv与read功能相同
  • flags=MSG_OOB，接收带外数据
  • flags=MSG_DONTWAIT，套接字缓冲区没有数据时，进程不阻塞等待
  • flags=MSG_WAITALL，通知内核直到读到请求的数据字节数时，才返回
  • flags=MSG_PEEK，接收数据时不从缓冲区移走数据，下一个读操作将会读到同样的数据

说明

既可以用于 TCP 也可以 UDP，但个人感觉用于 TCP 较好，需先 connect。比 read 功能强大，拥有 flag 参数

recvfrom

flag 参数

  • flags=0, recv与read功能相同
  • flags=MSG_OOB，接收带外数据
  • flags=MSG_DONTWAIT，套接字缓冲区没有数据时，进程不阻塞等待
  • flags=MSG_WAITALL，通知内核直到读到请求的数据字节数时，才返回
  • flags=MSG_PEEK，接收数据时不从缓冲区移走数据，下一个读操作将会读到同样的数据

说明

只能用于 UDP 的无连接通讯，一般只声明地址，内核自动填充地址

sendmsg & recvmsg

  • 功能：发送和接收规格化的数据，配对使用
  • 返回值：≥0—成功，-1—失败
  • 参数：
    • fd－socket描述符
    • msghdr－规格化数据缓冲区
    • flags－控制参数

writev & readv

  • 功能：一次读、写多个非连续缓冲区
  • 返回值：≥0—成功，已读写字节数；-1—失败
  • 参数：
    • fd－socket描述符
    • iov－缓冲区数组
    • iovcnt－缓冲区数组个数

close

说明

调用 close 后本进程将不能再使用这个套接字，但 TCP 可能没有删除套接字结构，因为其他进程可能还在使用。调用 close 只是对 sockfd 的引用-1，直到对 sockfd 的引用为 0 时才清除 sockfd。

shutdown

参数

  • sockfd－socket描述符
  • howto－指定关闭操作的类型
    • 0：停止读
    • 1：停止写
    • 2：停止读、写

说明

shutdown 关闭连接通道，所有进程不能再使用已被关闭的通道
howto=0，关闭读通道，丢弃尚未读取的数据，对后来接收到的数据返回确认后丢弃。
howto=1，关闭写通道，继续发送发送缓冲区未发送完的数据，然后发送FIN字段关闭写通道。
howto=2，关闭读写通道，任何进程不能再操作这个 socket。

close 与 shutsown

  • 关闭对象和影响范围不同：close 关闭本进程的 sockfd，但链接仍然开着，引用这个 sockfd 的其它进程还能用这个链接。
  • 关闭粒度不同：close 关闭进程与 socket 的整个接口，调用 close 之后本地进程不能再读写这个 socket ；shutdown可以只关闭一个方向的通道，另一个方向的通道仍然可以操作。

getpeername

参数与返回值

  • fd－socket描述符
  • proaddr－存放地址信息的缓冲区的指针
  • addrlen－存放缓冲区字节数的整型变量的指针
  • 返回值：0－成功，－1失败

说明

功能：获得对端socket的地址信息。当一个调用 accept 的进程通过 fork 和 exec 启动一个新的进程时，经常要调用这个函数。新的服务器通过getpeername函数来得到远端的地址。通常，要在应用的访问地址表查找返回地址，如果返回地址不在访问表中，则连接将被关闭。

getsockname

参数与返回值

  • fd－socket描述符
  • proaddr－存放地址信息的缓冲区的指针
  • addrlen－存放缓冲区字节数的整型变量的指针
  • 返回值：0－成功，－1失败

说明

功能：获得本地socket的地址信息。当在bind调用中使用了通配符地址INADDR_ANY时，该函数非常有用。

getsockopt

参数

  • sockfd－socket描述符
  • level－选项级别
    • SOL_SOCKET：通用socket选项
    • IPPROTO_TCP：TCP选项
    • IPPROTO_IP：IP选项
  • optname—选项名称
  • optval—选项值指针
  • optlen—选项值的长度/存放选项值长度的指针

示例

通用 socket 选项都有啥

  • SO_KEEPALIVE：设置该选项后，一段时间（通常2小时）内没有数据交换时，TCP协议将自动发送探测数据包，检查网络连接。
  • SO_RCVBUF和SO_SNDBUF：发送和接收数据缓冲区的大小（在连接建立以前设置）。
  • SO_RCVTIMEO和SO_SNDTIMEO：发送和接收超时，当指定时间内数据没有成功接收或发送，发送和接收函数将返回。
  • SO_REUSEADDR：一般一个端口释放后会等待两分钟才能被再次使用，该选项可让端口释放后立即就可以被再次使用，这通常在重启监听服务器时可避免 bind 函数出错

TCP选项都有啥

  • TCP_MAXSEG－TCP协议最大数据段长度
  • TCP_NODELAY－小数据包是否延迟发送（Nagle算法）

IP选项都有啥

  • IP_HDRINCL－是否在数据缓冲区中包括IP首部，适用于原始socket
  • IP_OPTIONS－IP首部选项
  • IP_TOS－服务类型与优先级
  • IP_TTL－生存时间

fcntl

  • 功能：改变套接字属性
    • 比如设置socket为阻塞/非阻塞模式
    • 设置允许/不允许接收异步I/O信号
    • 设置/获取socket的所有者
  • 参数与返回值
    • fd－文件（socket）描述符
    • cmd－执行的操作
    • 其他参数－根据cmd选择适当参数
    • 返回值：≥0－成功，-1－失败

ioctl

  • 功能：控制输入输出
  • 参数与返回值
    • fd－文件（socket）描述符
    • req－执行的操作类型
    • 第三个参数－总是指针类型，存储操作返回的数据或操作所需的数据
    • 返回值：0－成功，-1－失败
