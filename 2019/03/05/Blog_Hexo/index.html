Hexo 目录结构说明

  • config.yml
    • 博客的配置文件
    • 名称、关键词、作者、语言、博客主题等
  • package.json
    • 应用程序信息，新添加的插件内容也会出现在这里面
    • 可以不修改这里的内容。
  • scaffolds
    • scaffolds就是脚手架的意思，这里放了三个模板文件
    • 新添加博客文章 posts
    • 新添加博客页 page
    • 新添加草稿 draft 的目标样式
    • 我们可以在模板上添加比如 categories 等自定义内容
  • source
    • source是放置我们博客内容的地方，里面初始只有两个文件夹
    • drafts 草稿 和 posts 文章 ，
    • 通过命令新建tags 标签 还有categories 分类 页后，这里会相应地增加文件夹。
  • themes
    • 放置主题文件包的地方
    • Hexo会根据这个文件来生成静态页面。

上传 Github

不能直接 git-push

血泪教训！Hexo 与 Jekyll 不同，Github 原生支持 Jekyll，因此直接把 Jekyll 的文件扔上去就成。但是 Hexo 先要本地编译，再把编译后的东西上传
因此不能无脑 git push，需要用 hexo 的命令 deploy。
