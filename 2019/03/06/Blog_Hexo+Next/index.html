Hexo + NexT(v5和v6+)功能扩建与美化实录

发表于 2019-03-06 | 更新于 2019-03-19 | 分类于 Blog | 评论数： | 阅读次数：
本文侧重 Hexo+NexT 的站点功能扩建与美化。包括添加分类与标签；使用图床与图床工具；添加评论；添加游客统计；添加或更换背景图片、配色、头像、图标、动画效果；自定义界面等等。特别强调：本文区分 NexT 版本(V5 和 V6+ 包括 v7)，针对不同版本有不同的配置方法。

NexT 的 V5 与 V6+ 版本区别

这里不长篇大论了，只能说还是方便了一些。这里只列举三个：

  1. 侧边栏头像：V6 +侧边栏头像可以直接去注释实现圆形和鼠标悬停旋转；V5 需要加 css，比较麻烦。
  2. Github Banner：V6+可以直接去注释实现添加 Github 小上标；V5 需要添加代码。
  3. 解决 leancloud 文章阅读数量统计和 Valine 评论的冲突问题！在 V5 版本下，同时使用基于 leancloud 的阅读统计和 valine 评论是不可以的，有一个一定会失效，听说可以解决，但很麻烦。V6解决了这个问题，并且配置变得相当简单。

想要标签与分类界面

  • 去掉 menu 注释
    在 next 主题目录下寻找 config.yml ,找到 menu 栏去掉注释。

    • 配置位置：/theme/next/_config.yml

  • 创建页面
    hexo new page 命令进行页面创建,主题的 _config.yml 文件中的 menu 中进行匹配，执行完后再/source 中出现 categories 和 tags 两个新目录，里面分别有一个 index.md。

  • 添加 type 关键字
    上一步之后你会发现 categories 和 tags 页面出来了，但是没东西，侧边栏也没有链接。于是乎要在新建的 categories 和 tags 目录下的 index.md 中添加 type 关键字。
    • 配置位置：/source/categories/index.md 和 /source/tags/index.md

背景图片

插入本地固定图片

配置位置：themes\next\source\css\ _custom\custom.styl

将一张图片 (推荐大小1600x900) 放入/themes/next_v6/source/images/中，我的命名为 bg.jpg。修改 custom.styl 在 url 位置填入 /images/bg.jpg

  • background:url 图片的地址
  • repeat：是否重复出现
  • attachment：定义背景图片随滚动轴的移动方式
  • position：设置背景图像的起始位置

github Banner

###V6想要 Github 跳转上标？

相比 V5 ，V6 可是相当容易。

配置位置：/theme/next/_config.yml 的 github_banner 字段
将 enable 设为 true，链接设为自己的即可。

V5想要 Github 跳转上标？

配置位置：/themes/next/layout/_layout.swig 文件中 <div class="headband"></div>下
修改 https://your-url 、fill:#151513; 和 color:#fff;
注意这个 your-url 是你的 github 的 url

底部井号改为标签

修改模板 /themes/next/layout/_macro/post.swig，搜索 rel="tag">#，将 # 换成<i class="fa fa-tag"></i>

这个需要重新 hexo g 生成一下，然后才能见效。
