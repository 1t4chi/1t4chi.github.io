Jekyll + Github 从入门到死亡

发表于 2019-02-13 | 更新于 2019-03-15 | 分类于 Blog | 评论数： | 阅读次数：
想着把以前的笔记上Blog，无意惊扰了 Jekyll，苦海无涯任我漂。

jekyll 是什么

Jekyll (杰基尔)是一个生成静态网页的工具，得到 Github 的原生支持。你一上来 Github Page 人家就让你选 Jekyll Theme，会误导很多人觉得 Jekyll 是最好的选择。实际上，Jekyll 是很好，但是对于不熟悉 ruby、前端开发的人来说简直是噩梦。因为它的主题库相比之下要简陋很多，页面的动态刘畅感也无法让人满意。对于前端开发的高手来说，可能更喜欢 Jekyll，它像一张白纸，会画的人可以画的精彩绝伦，但不会画的人也只有欲哭无泪了。

因此我最后的 Jekyll 站布的并不成功。虽然搭起来了，但是看上去总是像 20 世纪初的产物，花花绿绿又十分僵硬。可惜所有的花花绿绿图都是我自己做的，做了半天嘤嘤嘤。现在我已经全面搬迁 Hexo，好搭了不是一点半点，界面还美观了一些。有兴趣的可以看看我写的几篇关于 Hexo 的文章。

问题与解决

P1: Could not locate Gemfile or .bundle/ directory

下述命令有时会报错。因为 bundle exec 会在当前项目依赖的上下文环境中执行命令：相同的 gem、相同的依赖关系、相同的版本号

  • bundle 是啥？
    bundle 是用来管理项目的 gem 的。没有 Gemfile 不能加 bundle。所以当直接运行一个已存在的站点时不用加 bundle exec。
